## Context

### Original Request

User wanted to replicate `playgrounds/vue3/` into a new Rspack playground and fix audio issues described in `docs/AUDIO_PLAYBACK_FLOW.md`. Investigation revealed the root cause is a simple bug in the existing Vite playground's monkey-patch, making the Rspack playground unnecessary.

### Interview Summary

**Key Discussions**:

- Root cause confirmed: monkey-patch patches `AudioContext.prototype.addModule` but `addModule` lives on `AudioWorklet.prototype` (inherited from `Worklet.prototype`)
- The SDK (webpack worklet-loader, inline: true) creates blob: URLs for worklets. Blob: URLs can't resolve relative ES module imports (`import opus from './opus'`), so worklets fail silently
- The IIFE worklet files + `opuslibs.wasm` are already in `public/` — they just never get loaded because the redirect never fires
- User chose smallest scope: fix Vite playground only

**Research Findings**:

- Explore agent: Confirmed failure chain — blob URL → import fails → no OPEN_RSP → 5s timeout → "请求超时 1"
- Librarian: Blob URLs DO work with `addModule()` in browsers, but internal relative imports break. CSP `worker-src 'self' blob:` may be needed but monkey-patch bypasses blob URLs entirely
- Metis identified a SECOND bug: `moduleURL.split('/').pop()` on a blob URL like `blob:https://localhost:5173/uuid` extracts `uuid`, not a worklet filename — so the workletMap lookup always fails even if the patch executed

### Metis Review

**Identified Gaps** (addressed):

- TWO bugs not one: prototype target AND URL-matching logic both broken — Both fixed in Task 1
- Unused Vite plugins (`vite-plugin-weila-sdk.ts`, `vite-plugin-audio-worklet.ts`) sitting in repo — Cleanup in Task 2
- `docs/AUDIO_PLAYBACK_FLOW.md` perpetuates the wrong prototype info — Fixed in Task 2

    // Solves Vite + webpack worklet-loader (inline: true) incompatibility:
    // SDK creates blob: URLs for worklets, but blob URLs can't resolve
    // relative ES module imports (import opus from './opus'), causing silent failure.
    // We redirect to pre-built IIFE versions in public/ that have no imports.
    const originalAddModule = AudioWorklet.prototype.addModule
    const workletRedirects = ['/weila_player.worklet.iife.js', '/weila_recorder.worklet.iife.js']
    let workletCallIndex = 0

    AudioWorklet.prototype.addModule = async function (moduleURL: string, ...args: any[]) {
      // Blob URLs from webpack worklet-loader can't resolve internal imports
      if (moduleURL.startsWith('blob:') && workletCallIndex < workletRedirects.length) {
        const redirectTarget = workletRedirects[workletCallIndex++]
        console.log(
          '[Weila] Redirecting worklet:',
          moduleURL.slice(0, 40) + '...',
          '->',
          redirectTarget,
        )
        return originalAddModule.call(this, redirectTarget, ...args)
      }

      // Fallback: filename-based matching for non-blob URLs
      const workletMap: Record<string, string> = {
        'weila_player.worklet.js': '/weila_player.worklet.iife.js',
        'weila_recorder.worklet.js': '/weila_recorder.worklet.iife.js',
      }
      const fileName = moduleURL.split('/').pop()
      if (fileName && workletMap[fileName]) {
        console.log('[Weila] Redirecting worklet:', fileName, '->', workletMap[fileName])
        return originalAddModule.call(this, workletMap[fileName], ...args)
      }

      return originalAddModule.call(this, moduleURL, ...args)
    }
